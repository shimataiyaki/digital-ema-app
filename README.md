# digital-ema

願い事をリアルタイムで共有できる、文化祭向けデジタル絵馬アプリ

## 概要
digital-emaは、来場者が入力した願い事を「デジタル絵馬」としてリアルタイムに表示・共有できるWebアプリです。  
大型モニターでのライブ表示と、スマートフォンからの投稿を組み合わせ、**会場全体で体験を共有する設計**になっています。

## 特徴
- 願い事をリアルタイムで表示するデジタル絵馬
- 大型モニターでのライブ表示（展示向け最適化）
- 新規投稿のリアルタイム通知
- 当日表示＋アーカイブの二層構造
- スマートフォンから簡単に奉納可能

## 使い方（来場者向け）
1. QRコードまたは端末からフォームにアクセス  
2. ニックネームと想いを入力して奉納  
3. 大型モニターに表示されるのを待つ  
4. 他の来場者の想いも閲覧する  

## システムの流れ
```
[来場者のスマホ] → [Googleフォーム] → [スプレッドシート]
↓
[GASでデータ取得]
↓
[大型モニターに表示]
```

## 技術構成
- **フロントエンド**：HTML5 / CSS3 / JavaScript（ES6）
- **バックエンド**：Google Apps Script（GAS）
- **データベース**：Google スプレッドシート
- **フォーム**：Google フォーム
- **ホスティング**：GitHub Pages

## 技術解説

### データフロー
- フォーム入力 → スプレッドシートに保存  
- GASが定期取得し、フロントへJSONとして配信  
- フロント側で描画・更新  

### 表示ロジック
- 当日データのみを抽出
- 最大15枚をランダム表示（30秒ごと更新）
- アーカイブは全件を時系列表示（45秒更新）

### UI/UX設計
- 縦書きの絵馬デザイン
- リアルタイム通知による参加感の強化
- モバイル・大型ディスプレイ双方に最適化

## 運用設計（コア設計）
- 投稿 → 即反映ではなく、一定間隔で更新
- 表示数制限により視認性を維持
- 通知で「自分の投稿が届いた」実感を演出

▶単なる掲示板ではなく、**体験としての“奉納”を設計している**

## セットアップ方法

1. Googleフォームを作成  
   - ニックネーム（記述式）  
   - 想い（段落）  
2. スプレッドシートと連携  
3. Apps Scriptでプロジェクト作成  
4. 本リポジトリのコードをコピー  
5. `SPREADSHEET_ID` を書き換え  
6. Webアプリとしてデプロイ（全員アクセス可）  

## カスタマイズ
| 内容 | 箇所 |
|:---|:---|
| 表示枚数 | `DISPLAY_COUNT` |
| 更新間隔 | `setInterval` |
| 通知人数 | `getRecentNicknames()` |
| 文字数制限 | `maxLen` |

## 開発背景
本プロダクトは、文化祭における「神社」展示のために開発されました。

- 来場者の参加を促す仕組み  
- 会場全体で共有できる体験  
- 運用負荷を抑えた構成  

これらを両立するため、  
**GAS + スプレッドシートによる軽量なリアルタイムシステム**として設計されています。
